# Git冒险 - 交互式Git学习游戏

> [!note]
> 本项目完全由AI生成

## 项目概述

Git冒险是一个基于Web的交互式游戏，专为Git初学者设计。通过完成一系列精心设计的关卡任务，用户可以在浏览器中模拟的Git环境里直观学习和练习Git命令，无需安装任何软件。每个关卡都提供了清晰的目标和即时反馈，帮助用户循序渐进地掌握Git技能。

## 功能特点

- **渐进式关卡系统**：从Git基础命令到高级操作，系统地学习Git
- **交互式终端环境**：在浏览器中直接输入Git命令并查看执行结果
- **即时验证与反馈**：实时检查命令正确性，提供清晰的错误提示
- **响应式设计**：适配各种屏幕尺寸，提供流畅的游戏体验
- **主题切换**：支持深色和浅色两种显示模式
- **命令参考系统**：内置完整的命令列表和使用示例
- **直观的关卡完成界面**：通过模态窗口展示完成信息和返回选项

## 技术栈

- **HTML5** - 构建页面结构
- **CSS3** - 实现样式和主题
- **JavaScript** - 处理游戏逻辑和交互
- **GitHub Actions** - 自动化部署流程

## 核心命令支持

游戏支持以下Git及相关命令：

- `git init` - 初始化Git仓库
- `git add` - 将文件添加到暂存区（支持单个文件或 `git add .` 添加全部）
- `git commit` - 创建提交（支持 `-m` 指定提交信息，支持 `-a` 自动暂存修改文件）
- `git status` - 查看工作区状态
- `git log` - 查看提交历史
- `ls` - 列出当前目录内容
- `cat` - 查看文件内容
- `echo` - 创建或修改文件内容
- `mkdir` - 创建新目录
- `help` - 显示基本帮助信息

## 游戏机制详解

### 关卡系统

- 每个关卡都有明确的学习目标和任务描述
- 用户需要通过输入Git命令来解决特定问题
- 完成关卡后，会弹出模态窗口提示成功，并可返回关卡选择
- 关卡进度会在页面上直观显示

### 命令执行机制

- 游戏模拟了真实Git环境的基本行为
- 提供了智能的命令解析，可以理解常见的Git命令格式
- 对于无效命令会提供友好的错误提示和建议

## 开发与扩展

### 添加新关卡

1. 参考 `关卡配置指南.md` 中的格式要求
2. 在 `levels.js` 文件中添加新的关卡配置
3. 配置包含关卡ID、标题、描述、初始状态和完成条件

### 添加新命令

1. 参考 `命令描述指南.md` 中的实现方法
2. 在 `game.js` 的命令系统中添加新命令的处理逻辑
3. 确保提供清晰的描述和示例

## 自动部署

项目已配置GitHub Actions工作流，实现代码推送后自动部署到GitHub Pages：
- 推送代码到 `main` 或 `master` 分支时自动触发
- 部署到 `gh-pages` 分支的 `pages/LearnGit` 目录
- 无需手动干预，简化发布流程
